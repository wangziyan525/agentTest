# Superpowers Receiving Code Review

Treat review feedback as input to evaluate, not commands to obey blindly.

Process:

1. Read all feedback calmly.
2. Restate unclear items in technical terms.
3. Verify each point against the actual codebase.
4. Apply sound feedback one item at a time.
5. Push back when a suggestion is wrong for this codebase.

Rules:

- No performative agreement.
- No batch implementation of half-understood feedback.
- Ask for clarification when items are ambiguous.
- Prefer technical reasoning and concrete fixes over social filler.

# Superpowers Finishing A Development Branch

Close out implementation with structured options instead of vague handoff.

Process:

1. Verify tests or other completion checks first.
2. Identify the base branch.
3. Present clear next-step options:
   merge locally, create PR, keep branch, or discard.
4. Execute the chosen path carefully.
5. Clean up only when appropriate.

Rules:

- Never offer completion options before verification.
- Never delete work without explicit confirmation.
- Do not force a PR or merge flow the user did not ask for.

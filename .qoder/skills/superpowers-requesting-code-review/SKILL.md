# Superpowers Requesting Code Review

Add a deliberate review checkpoint before problems compound.

Process:

1. Identify what changed and what the intended behavior is.
2. Gather the comparison range or files to review.
3. Request review against requirements, not just style.
4. Treat Critical issues as blockers.
5. Fix Important issues before continuing whenever practical.

Keep the review request focused:

- what changed
- what it should do
- what scope to inspect
- what output format is desired

# Superpowers Brainstorming

Use this before implementation when we should refine the idea first instead of jumping into code.

Process:

1. Explore the current project context first.
2. Ask clarifying questions one at a time.
3. Offer 2-3 approaches with tradeoffs and a recommendation.
4. Present the design in a proportionate, easy-to-review form.
5. Get user approval before moving to implementation.
6. If implementation is next, switch to `superpowers-writing-plans`.

Guardrails:

- Do not start coding during brainstorming.
- Do not bury the user in many questions at once.
- Stay grounded in the existing codebase and project constraints.
- If the task is actually small and already clear, keep the design short rather than skipping the step.

# Superpowers Writing Skills

Treat skill authoring like process-focused TDD.

Process:

1. Identify the repeated workflow or judgment pattern.
2. Define when the skill should trigger.
3. Write a concise `SKILL.md` focused on decisions and process.
4. Test whether the skill would actually change behavior.
5. Refine it until the trigger and guidance are clear.

Rules:

- Write the description as a trigger condition, not a process summary.
- Prefer reusable guidance over one-off storytelling.
- Keep the skill concise enough to be discoverable and readable.
